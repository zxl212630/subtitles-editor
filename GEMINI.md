# GEMINI.md

本文件为 Gemini CLI 提供项目特定的运行指导。

> ℹ️ **单一事实来源**：有关本项目详细的**架构设计**、**组件定义**、**构建命令**及**UI 规范**，请严格参考 [CLAUDE.md](./CLAUDE.md)。

## 运行指导原则

- **思考语言约束**：在进行思维链（Chain of Thought）以及内部规划、拆解任务时，**必须完全使用中文进行思考**。
- **回复语言约束**：所有最终输出、代码注释和错误分析，一律使用中文进行回复（代码本身除外）。
- **技术栈偏好**：本项目为纯 C++17/Qt6 Widgets 应用，禁止引入 QML 或非必要的第三方库。

## 核心任务逻辑

- **构建指令**：优先使用 `cmake --build cmake-build-debug`。
- **测试指令**：如果存在测试，使用 `ctest` 或运行 `build/tests/` 下的特定二进制文件。
- **样式规范**：严格遵守 `CLAUDE.md` 中定义的 UI 布局原则。

## 运行时调试与交互规范

1. **日志收集**：在启动程序进行调试时，优先使用 `nohup` 模式，将输出重定向到项目根目录下的 `startup.log` 或 `output.log`。
   - 示例：`nohup ./cmake-build-debug/subtitles-editor > startup.log 2>&1 &`
2. **视觉反馈（截图）**：当需要查看 UI 表现或错误弹窗时，应主动引导用户。调用系统截图工具（如 macOS 上的 `screencapture -ui`）进入交互式截图模式。
   - 步骤：提示用户“请在弹出的截图工具中选择相关区域”，然后等待用户操作完成。
3. **闭环排查工作流**：
   - **分析**：程序运行或用户操作后，立即读取并分析 `startup.log`。
   - **诊断**：结合日志中的 Qt 警告、FFmpeg 报错或自定义 `qDebug()` 输出，对照代码逻辑定位问题。
   - **验证**：修复后重新执行上述流程，确保日志中不再出现异常。
