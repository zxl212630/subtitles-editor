# 动态字幕背景图与说话人管理功能设计文档

**文档日期**: 2026-05-23  
**相关特性**: 说话人识别（Diarization）、动态背景图、九宫格缩放与固定长度绘制、统一素材文件夹、统一九宫格四向边距、独立说话人管理窗口、字幕列表快速切换、动态主题实时更新

---

## 1. 需求背景

在多说话人视频中，我们需要根据当前说话的人是谁，在视频中为字幕贴上不同的**动态背景图**。
1. **ASR 阶段**：自动检测说话人。通过开启说话人分离参数，ASR 识别结果能按 `Speaker 0`, `Speaker 1` 标识区分说话段。
2. **渲染模式**：支持九宫格（9-patch）随文本拉伸以保留圆角边缘，和非九宫格固定长度模式。非九宫格模式下文字允许直接溢出。
3. **统一参数配置**：
   - **统一背景图文件夹**：提供一个全局的文件夹路径，所有背景图都存放在此文件夹内。各个说话人通过选择该目录下的图片文件名进行绑定。
   - **统一四向九宫格边距**：不再在每张图上配边距，也不使用单一数值。而是使用全局统一的左、右、上、下四个边的保留像素大小（Margins），供所有九宫格图共同使用。
4. **持久化简化**：由于目前系统没有“工程文件”概念，每个视频的说话人映射在重启或重新载入后可直接丢失（以后有工程设计再保存映射关系）。系统**当前仅需要持久化保存“全局统一设置”**（背景图文件夹路径、九宫格四向边距数值）。
5. **动态主题支持**：管理窗口必须与软件全局主题关联，无论是主题切换还是主色调微调，管理窗口的色彩及样式应**实时更新**。

---

## 2. 详细设计与交互流程

### 2.1 说话人管理窗口 (SpeakerManagerDialog)
基于 `QDialog` 并继承自 `QWK::WidgetWindowAgent` 实现，支持动态主题更新（在 `ThemeManager` 改变信号触发时，即时刷新 QSS 样式表）。

```
+-----------------------------------------------------------+
| ⚙️ 说话人管理                                             X |
+-----------------------------------------------------------+
| [统一设置]                                                |
| 背景图文件夹: [ /Users/zxl/Documents/字幕背景/ ]  [选择目录] |
| 九宫格边距:   左: [15]px  右: [15]px  上: [15]px  下: [15]px  |
+-----------------------------+-----------------------------+
| 说话人列表                  | 属性配置                    |
| +-------------------------+ | 说话人名称:  [ Alice     ]  |
| | Speaker 0 (Alice)       | |                             |
| | Speaker 1 (Bob)         | | 图片文件名: [ alice.png v ]  |
| |                         | | +-------------------------+ |
| |                         | | |     (图片缩略图预览)     | |
| |                         | | +-------------------------+ |
| |                         | | 绘制模式:    [ 九宫格缩放  v]  |
| +-------------------------+ +-----------------------------+
| [ + 新增 ]   [ - 删除 ]     |                             |
+-----------------------------+-----------------------------+
|                                         [保存并应用] [取消] |
+-----------------------------------------------------------+
```

1. **顶部 - 统一参数设置区（统一设置）**：
   - 位于标题栏下方、说话人列表/配置的上方。
   - **背景图文件夹**：一个显示路径的 `QLineEdit` 与 `[选择目录]` 按钮。点击可调出文件夹选择器（`QFileDialog::getExistingDirectory`）。
   - **九宫格边距**：四个独立微调输入框 `QSpinBox`（分别对应左、右、上、下的保留像素大小，默认值均为 15px）。
2. **左侧 - 说话人列表**：
   - 使用 `QListWidget` 列出当前关联的所有说话人。
   - 双击列表项即可直接在行内重命名名称（双击时激活编辑状态）。
   - 底部包含 `[+ 新增]` 与 `[-] 删除` 按钮，用于添加新的或删除说话人。
3. **右侧 - 说话人属性配置**：
   - **名称编辑**：同步修改选中的说话人名字。
   - **图片文件名**：一个 `QComboBox` 下拉框。当在顶部“背景图文件夹”选中目录后，系统**自动扫描该文件夹**下的所有可用图片格式（如 `*.png`、`*.jpg`），并填充进下拉框中。用户直接从中下拉选择该说话人对应的图片文件名称即可。
   - **缩略图预览**：下方置一 `QLabel`，当选中文件名后，拼接文件夹路径加载并显示该背景图的缩略图。
   - **绘制模式下拉框**：可选择 `九宫格缩放` 或 `固定长度`。
4. **底部 - 操作区**：
   - **[保存并应用]**：将说话人映射、全局边距和文件夹路径写入 `SubtitleTrack`；将全局统一设置（文件夹路径、四向边距值）写入本地配置文件 `config.ini` 中永久保存；向应用广播刷新信号，视频预览即时生效。
   - **[取消]**：丢弃本次修改，关闭窗口。

### 2.2 字幕列表内联切换 (方案 A 交互)
在左侧的 `SubtitleListPanel` 中：
1. **字幕行（SubtitleItem）高度保持 56px**：保持原有的高密度列表显示，不增加高度。在横向上通过新增一个“说话人列”来实现布局。
2. **表头与列表项完美网格对齐设计**：
   - 表头 `SubtitleTableHeader` 的 `hlLayout`（水平布局）统一将间距（spacing）调整为 `12px`。
   - 表头左侧的“时间码”标签 `headerTime_` 固定宽度设为 `100px`。
   - 在其右侧新增“说话人”表头标签 `headerSpeaker_`（显示文字 "Speaker"），固定宽度设为 `80px`。
   - “字幕文本”表头标签 `headerText_` 自动拉伸，占据右侧剩余空间。
3. **横向四列排版布局**：
   - **第一列：时间码**（起于列表项 `rect.left() + 12`，宽 `100px`），上下两行显示开始/结束时间，与表头的 `headerTime_` 完美垂直对齐。
   - **第二列：说话人标签（新增）**（起于列表项 `rect.left() + 124`，宽 `80px`），在其中央区域绘制一个圆角的半透明药丸标签（使用 `ThemeManager::getPrimaryColor()` 填充），显示说话人名称（如 `Alice` 或 `Speaker 0`），若未分配则显示 `未分配`。在位置和宽度上与表头的 `headerSpeaker_` 完美对齐。
   - **第三列：字幕文本**（起于列表项 `rect.left() + 216`，终点为操作按钮左侧 `rect.right() - 60`），在此区域内绘制/编辑字幕文本内容，与表头的 `headerText_` 完美对齐。
   - **第四列：操作按钮**（宽 `36px`，靠右对齐），显示剪刀与删除图标。
4. **下拉菜单交互**：当用户点击字幕行上的说话人标签时，或者在双击激活编辑时，在该位置弹出一个 `QMenu`。
   - 菜单项列出当前配置的所有说话人名字。
   - 菜单底部提供 `[+ 新建说话人...]` 与 `[⚙️ 管理说话人...]` 选项。
   - 用户选择其中一个说话人，将直接修改该条字幕的 `speakerId`。若点击管理，则弹出上述 `SpeakerManagerDialog`。

## 3. C++ 接口与逻辑设计

### 3.1 ASR 与数据模型扩展
1. **TencentAsrService 扩展**：
   - `payload()` 构造函数中，添加 `"SpeakerDiarization": 1` 参数。
   - 在 `parseResultDetail` 回调中，从返回句子的 JSON 中解析出 `"SpeakerId"`（腾讯云 ASR 返回值为整数），并填充给 `TranscriptSegment::speakerId`。
2. **SubtitleItem 扩展**：
   - 在 `SubtitleItem` 结构体中添加 `int speakerId = -1;` 属性。
3. **SubtitleTrack 数据流维护**：
   - 增加说话人相关结构体及属性：
     ```cpp
     struct SpeakerInfo {
       int id = -1;           // 说话人 ID (从0开始的整数)
       QString name;          // 说话人名称
       QString bgImageFile;   // 背景图图片文件名 (例如 "alice.png"，放在统一目录下)
       bool is9Patch = true;  // 是否九宫格缩放
     };
     ```
   - 维护 `QMap<int, SpeakerInfo> speakers_`。
   - 维护统一设置：
     - 背景图文件夹路径：`QString globalBgFolder_`。
     - 统一九宫格边距：`QMargins unifiedBorderMargins_`（默认左 15, 右 15, 上 15, 下 15）。
   - 增加统一设置的 Getter/Setter：并在 `setUnifiedSettings` 时把路径和边距存入 `config.ini`（使用 `QSettings` 全局持久化）。
   - 增加 `signals: void speakersChanged();` 信号。

### 3.2 动态主题实现细节
- `SpeakerManagerDialog` 连接全局 `ThemeManager` 的 `themeChanged()` 信号。
- 在 `updateTheme()` 函数中，实时读取当前的全局配色：
  ```cpp
  QColor bgPanel = ThemeManager::instance().getBgPanelColor();
  QColor textNormal = ThemeManager::instance().getTextNormalColor();
  QColor primary = ThemeManager::instance().getPrimaryColor();
  // 组装并应用新样式表
  setStyleSheet(QString("QDialog { background-color: %1; ... }").arg(bgPanel.name()).arg(...));
  ```

## 4. 背景图绘制算法 (SoftwareVideoRenderer)

绘制将在 `SoftwareVideoRenderer::paintEvent` 绘制字幕文字的步骤前执行：

1. **测算文字包围矩形**：
   使用当前配置的 `QFont` 及 `QFontMetrics::boundingRect` 计算出待绘制文本的实际宽高矩形。
   - 左右各外扩 20 像素 padding，上下各外扩 10 像素 padding，获得目标背景图矩形 `targetBgRect`。
2. **定位背景图片**：
   - 从 `SubtitleTrack` 中获取 `globalBgFolder_` 以及说话人关联的 `bgImageFile`。
   - 拼接得到物理文件路径：`QDir(globalBgFolder_).filePath(bgImageFile)`。
   - 若存在，优先从 `bgCache_`（`QHash<QString, QImage>`）提取缓存数据，未命中的图片读取到内存并加入缓存。
3. **如果为九宫格缩放 (9-Patch)**：
   - 将原图按照 `unifiedBorderMargins_` (左、右、上、下四个独立值) 分割成 9 个区块。
   - 4 个角（左上、右上、左下、右下）保持原像素宽高比例不变，直接绘制在 `targetBgRect` 的 4 个角上。
   - 4 个边（上、下、左、右）在对应的单轴上进行拉伸绘制以填满空间。
   - 中间区域在双轴上进行拉伸绘制填充。
4. **如果为固定长度 (不缩放)**：
   - 保持背景图片的原始分辨率（`bgImage.size()`）。
   - 将该图片居中绘制在文字正下方。
   - **如果文字的总宽度超出了背景图片的宽度，直接绘制溢出部分，不做任何截断和折行**。
