# 配置窗口设计文档

本文档描述了字幕编辑器中原生桌面配置对话框的实现设计，以及底层设置项（主题、语言）的配套实现方案。

## 1. 概述
新增一个配置窗口，允许用户管理通用设置（语言、主题）、存储提供商（OSS）和 ASR 提供商（腾讯云等）。目标是提供一个专业、原生且与现有深色主题一致的 UI。

## 2. UI 架构
- **组件**: `ConfigDialog` (继承自 `QDialog`)。
- **布局**: 
  - **左侧边栏**: `QListWidget` 用于分类选择（通用、存储、ASR）。
  - **右侧内容区**: `QStackedWidget` 包含各分类对应的页面。
  - **底部页脚**: 常驻区域，显示状态文本（“未保存的更改”）及按钮（确定、取消、应用）。
- **主题**: 深色模式（背景色 #1e1e1e），与主窗口 `AppWindow` 保持一致。

## 3. 核心功能及配套实现

### 3.1. 通用配置配套实现
由于项目目前尚不支持主题和语言切换，我们将采用以下方案：

#### 3.1.1. 主题切换 (Theme Switching)
- **方案**: 采用动态加载 QSS 样式表。
- **实现**:
  - 在 `ConfigManager` 中记录 `theme` 字段（如 "dark", "light"）。
  - 创建独立的 `themes/dark.qss` 和 `themes/light.qss`（目前主窗口的样式将提取到 dark.qss）。
  - 在 `AppWindow` 中实现 `applyTheme(QString theme)` 方法，读取对应文件并调用 `qApp->setStyleSheet()`。
- **首期任务**: 仅提取现有深色样式到独立文件，预留浅色主题接口。

#### 3.1.2. 语言切换 (Language Switching)
- **方案**: 使用 Qt 标准的 `QTranslator` 机制。
- **实现**:
  - 使用 Qt Linguist 生成 `.ts` 文件（如 `zh_CN.ts`, `en_US.ts`）。
  - 在 `main.cpp` 中根据 `ConfigManager` 的语言配置加载 `.qm` 文件。
  - **动态切换**: 切换语言后，提示用户“重启程序以生效”（这是最稳妥的桌面端处理方式），或通过 `changeEvent` 处理 `QEvent::LanguageChange` 实现实时切换（可选）。
- **首期任务**: 建立翻译文件架构，支持中/英切换界面文案。

### 3.2. 存储/ASR 配置
- 顶部 `QComboBox` 用于选择服务提供商。
- 表单字段根据选定的提供商动态更新。
- “测试连接”按钮用于在保存前验证凭据。

### 3.3. 脏数据管理 (方案 B)
- 对话框通过比对当前 UI 值与原始加载配置来跟踪变更。
- **状态页脚**: 当存在变更时，以黄色显示 “● 您有未保存的更改”。
- **保存流程**:
  - `应用 (Apply)`: 将当前状态保存到 `ConfigManager` 而不关闭窗口，并触发主题/语言的实时应用（如果支持）。
  - `确定 (OK)`: 保存并关闭窗口。
  - `取消 (Cancel)`: 放弃修改并关闭（若有变更则弹出提示）。

## 4. 技术细节
- **提供商切换**: 在存储/ASR 页面内使用子 `QStackedWidget` 来切换特定提供商的表单。
- **敏感数据**: 对 Secret Key 使用 `QLineEdit::Password` 模式。
- **验证**: “测试连接”将触发异步网络请求，并显示成功/失败状态。

## 5. 视觉资源
- **设置图标**: 在标题栏添加 24x24 的 SVG 齿轮图标。

## 6. 验收标准

- [ ] 分类切换功能正常。
- [ ] 更改提供商时能正确切换表单字段。
- [ ] “未保存的更改”提示在变更时能正确显示。
- [ ] 保存操作能真实更新 `config.ini` 文件。
- [ ] 主题切换能够加载不同的样式表（或预留文件）。
- [ ] 语言切换能够加载翻译文件（或预留架构）。
