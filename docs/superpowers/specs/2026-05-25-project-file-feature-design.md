# 字幕编辑器工程文件功能设计

## 概述

为字幕编辑器添加工程文件支持，允许用户保存和打开工程文件，同时添加菜单栏和配置页面。

## 功能需求

**文件扩展名：** `.sedit`

**数据存储：**
- 默认样式和说话人设置保存在 `config.ini` 中（全局设置）
- 工程文件中的 `settings` 字段保存当前项目的覆盖设置
- 打开工程时：先加载 `config.ini` 的全局设置，再用工程文件中的 `settings` 覆盖
- 保存工程时：只保存与全局设置不同的项目特定设置

### 4. 自动保存和快捷键

**自动保存：**
- 间隔：1 分钟
- 触发条件：字幕修改、说话人修改、全局设置修改
- 保存位置：与工程文件相同目录，备份文件名 `*.sedit.bak`

**快捷键：**
- macOS：`Cmd+S`、`Cmd+Shift+S`、`Cmd+O`、`Cmd+N`
- Windows/Linux：`Ctrl+S`、`Ctrl+Shift+S`、`Ctrl+O`、`Ctrl+N`

**最近文件：**
- 保存位置：`config.ini` 或单独的 JSON 文件
- 最多保存 10 个最近文件
- 支持清除最近文件列表

### 5. 错误处理

**文件操作错误：**
- 打开工程文件失败：显示错误信息，允许重试或取消
- 保存工程文件失败：显示错误信息，询问是否重试或另存为
- 视频文件找不到：显示警告，允许继续编辑字幕（但无法播放）

**版本兼容性：**
- 未知版本号：显示警告，尝试读取
- 缺少必需字段：使用默认值填充
- JSON 解析失败：显示错误信息

## 实现方案

### 方案 A：渐进式实现（推荐）

**步骤：**
1. 先实现工程文件保存/加载（JSON 序列化）
2. 添加菜单栏和标题栏
3. 实现"字幕设置"配置页面
4. 集成自动保存和快捷键

**优点：**
- 每个步骤可独立测试
- 风险可控，出问题容易定位
- 符合 YAGNI 原则

**缺点：**
- 需要多次修改 AppWindow

### 方案 B：完整一次性实现

**步骤：**
- 一次性实现所有功能：工程文件 + 菜单栏 + 配置页面 + 自动保存

**优点：**
- 代码结构更统一

**缺点：**
- 改动量大，风险高
- 难以测试和调试

### 方案 C：先配置后工程

**步骤：**
1. 先实现"字幕设置"配置页面
2. 再实现工程文件保存/加载
3. 最后添加菜单栏

**优点：**
- 配置页面可独立使用
- 逻辑更清晰

**缺点：**
- 工程文件依赖配置页面

## 推荐方案

**方案 A：渐进式实现**

理由：
1. 工程文件是核心功能，优先实现
2. 菜单栏需要工程文件功能来展示
3. 配置页面可以稍后添加

## 技术细节

### 动态主题和多语言支持

**所有新增 UI 组件必须支持：**
- 动态主题切换（Dark/Light/OLED/Midnight）
- 动态语言切换（中文/English）
- 响应 `QEvent::LanguageChange` 和 `QEvent::PaletteChange` 事件

**具体要求：**
1. 菜单栏：所有菜单项和快捷键提示需要使用 `tr()` 包装
2. 标题栏：文件名显示需要支持动态更新
3. 配置页面：所有标签和按钮需要支持 `retranslateUi()`
4. 错误提示：所有对话框文本需要使用 `tr()` 包装

### 数据结构

**SubtitleProject（新类）：**
- 负责工程文件的序列化和反序列化
- 管理工程元数据（版本、时间戳）
- 处理视频路径的相对/绝对路径转换

**ProjectManager（新类）：**
- 管理当前打开的工程
- 处理保存/加载操作
- 管理最近文件列表
- 处理自动保存逻辑

### 集成点

**AppWindow 修改：**
- 添加菜单栏和标题栏
- 添加快捷键支持
- 集成 ProjectManager

**ConfigDialog 修改：**
- 添加"字幕设置"页面
- 读取和保存全局设置

**SubtitleTrack 修改：**
- 添加序列化方法（toJsonObject / fromJsonObject）
- 添加修改状态跟踪
